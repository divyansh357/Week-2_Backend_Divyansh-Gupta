How Notification Service Works

üéØ Independent Microservice Architecture

Deployment: The notification service runs as a separate service on Render

  • API Gateway: https://ecommerce-api-gateway-dv80.onrender.com
  • Notification Service: Runs independently on different URL
  • Database: Shared PostgreSQL instance

üì® How It Works:

  1. User places an order via API Gateway
  2. API Gateway processes the order and saves to database
  3. API Gateway sends POST request to Notification Service
  4. Notification Service receives order details
  5. Email notification is logged (simulated)
  6. In production, this would use SendGrid/Nodemailer
